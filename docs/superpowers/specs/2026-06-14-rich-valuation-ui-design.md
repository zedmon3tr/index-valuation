# 丰富估值详情页设计

## 目标

在保持纯静态、零后端架构的前提下，将详情页改造成接近参考图的高信息密度分析界面，并加入可计算分位的历史股息率。

## 数据策略

- PE/PB：继续使用乐咕历史序列，中证 `index-perf` 作为 PE 兜底。
- A 股指数股息率：读取中证官方 `indicator/<code>indicator.xls` 中的“股息率2”，该字段按计算用股本统计，数据准确但只覆盖近期。
- 恒生指数股息率：使用乐咕 `api/stockdata/hs` 的 `dvRatio` 历史月度序列。
- 无可靠数据的指标不展示标签，不使用市场整体数据冒充具体指数数据。
- 每个指标显示样本量和覆盖日期，短历史不会伪装成长期数据。

## 详情页交互

- 指标：点位、PE、PB、股息率，按数据可用性动态展示。
- 时间：1 年、3 年、5 年、10 年、上市以来、自定义。
- 周期：日、周、月；按周期保留每个时间桶最后一条记录。
- 移动平均：无、20、60、120 期。
- 视图：统计分析与明细数据。
- 图层：分位线和标准差线可独立开关。
- 双轴：估值指标使用左轴，指数点位使用右轴；点位指标只使用单轴。

## 统计展示

左侧显示当前值、历史分位、70% 危险值、50% 中位数、30% 机会值、最大值、平均值、最小值、标准差、正负一倍标准差和 z 分数。

## 响应式

桌面端保持控制栏、统计栏和大图表的高密度布局。窄屏下控制栏横向滚动，统计与图表改为单列，明细表允许横向滚动。

## 验证

- 单元测试覆盖区间过滤、自定义日期、移动平均、周期采样、序列对齐和统计计算。
- Python 测试覆盖多数据源按日期合并。
- 浏览器验证沪深300和恒生指数详情页、控件状态、统计/明细切换和响应式布局。
